# Screenshot capture guide

This repo is set up to support consistent, portfolio-friendly screenshots in GitHub README files.

## Where to put images
- Store screenshots in `assets/screenshots/`
- Keep filenames stable so README links don't break

## Naming convention (recommended)
- `assets/screenshots/01-hero.png`
- `assets/screenshots/02-projects.png`
- `assets/screenshots/03-mobile.png`

## Recommended shots for this repo
1. Homepage hero section
2. Projects section with links
3. Mobile rendering (responsive view)

## Image format
- Prefer PNG
- Target ~1200px wide
